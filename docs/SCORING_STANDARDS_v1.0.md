## 一、技術指標（斷點5：metrics.py 技術面）

### 1.1 短線用指標

| 指標名稱 | 計算公式 | 說明 |
| --- | --- | --- |
| `MA_Alignment` | 布林值：`close > MA5 > MA10 > MA20` | 多頭排列判斷 |
| `Volume_Ratio` | `trading_volume / Vol_MA_5` | 量能是否放大，>1.2 視為明顯放量 |
| `RSI_6` | 6日 RSI（威廉指標常用短週期） | >80超買、<20超賣 |
| `Inst_5D_Net` | 外資+投信+自營商買賣超 `rolling(5).sum()` | 已在母表，此處直接讀取 |
| `Margin_5D_Change` | 融資餘額 `.diff(5)`，融券餘額 `.diff(5)` | 資券增減方向 |
| `Chip_Divergence` | 布林值：股價創新高但 `Inst_5D_Net` 為負，或反之 | 籌碼與價格背離警訊 |

### 1.2 波段用指標

| 指標名稱 | 計算公式 | 說明 |
| --- | --- | --- |
| `MA60_Bias` | `(close - MA_60) / MA_60` | 乖離率，判斷是否過度偏離中期均線 |
| `Revenue_Momentum` | 布林值：近3月 `Revenue_YoY_t > Revenue_YoY_{t-1} > Revenue_YoY_{t-2}` | 動能加速判斷 |
| `Inst_20D_Net` | 外資+投信+自營商買賣超 `rolling(20).sum()` | 中期籌碼方向 |
| `Big_Holder_Change` | 股權分散表中 `>1000張` 持股比例 `.diff(4)`（約1個月變動） | 大戶動向（若尚未串接股權分散表，此項可延後至擴充版） |
| `Price_Revenue_Divergence` | 布林值：股價創新高/新低，但 `Revenue_YoY` 走勢相反 | 營收股價背離警訊 |

## 二、財務指標（斷點6：metrics.py 財務面）

### 2.1 價值投資用指標

| 指標名稱 | 計算公式 | 說明 |
| --- | --- | --- |
| `ROE_TTM` | `TTM淨利 / 平均股東權益` | 已可由母表 TTM_EPS 系列衍生，或另計算 TTM淨利 |
| `ROE_Stability` | 近5年季度 ROE 的標準差 | 越低代表越穩定 |
| `Gross_Margin` | `毛利 / 營收`（當季） | |
| `Gross_Margin_Stability` | 近5年毛利率標準差 | |
| `PE_Percentile` | 直接讀取母表 | 已處理負值防呆 |
| `PB_Percentile` | 直接讀取母表 | |
| `Debt_Ratio` | `總負債 / 總資產` | |
| `FCF_vs_Dividend` | `TTM_FCF - 當年現金股利發放總額` | 正值代表現金流覆蓋股利 |

### 2.2 定存股用指標

| 指標名稱 | 計算公式 | 說明 |
| --- | --- | --- |
| `Dividend_Continuity_Years` | 近10年（或 Data_Years_Available）現金股利為0的次數是否為0 | 見 v2.1 定義：僅計現金股利 |
| `Payout_Ratio` | `當年每股現金股利 / 當年EPS` | |
| `Payout_Ratio_Stability` | 近10年 Payout_Ratio 標準差 | |
| `FCF_Coverage` | `TTM_FCF / 當年現金股利發放總額` | 覆蓋倍數，>1.5 通常視為安全 |
| `Debt_Ratio_Trend` | 近5年負債比線性回歸斜率 | 負值代表逐年下降（健康） |

## 三、四風格評分標準（斷點7：scorer.py）

四個風格各自輸出 0-100 分，由數個子項加總，子項權重總和為 100。以下權重為建議起點，可依你的偏好調整，但**建議 Cline 實作時把權重寫成可設定的 dict/config，而非寫死在程式裡**，方便後續微調不用改邏輯。

### 3.1 短線評分（100分）

| 子項 | 權重 | 給分規則 |
| --- | --- | --- |
| 均線多頭排列 | 30 | 完全滿足 `close>MA5>MA10>MA20` = 30分；滿足其中任3項排序 = 18分；任2項 = 8分；否則 = 0分 |
| 量能配合 | 25 | `Volume_Ratio ≥ 1.5` = 25分；`1.2~1.5` = 15分；`1.0~1.2` = 8分；`<1.0` = 0分 |
| 法人籌碼方向 | 25 | `Inst_5D_Net` 為正且創近5日新高 = 25分；為正 = 15分；為負 = 0分 |
| 資券健康度 | 20 | 融資增加+融券減少（籌碼健康）= 20分；`Chip_Divergence`為真（背離）= 0分；其餘 = 10分 |

**特殊情況**：RSI_6 > 85 時，無論以上加總多高，總分乘以 0.7 折扣（過熱風險警示，避免追高訊號被均線/量能蓋過）。

### 3.2 波段評分（100分）

| 子項 | 權重 | 給分規則 |
| --- | --- | --- |
| 營收動能 | 35 | `Revenue_Momentum` 為真（連續加速）= 35分；YoY為正但未加速 = 20分；YoY為負 = 0分 |
| 均線位置 | 25 | `close > MA_60` 且 `MA60_Bias` 在 0%~15% 之間（健康的多頭趨勢，非過熱）= 25分；`MA60_Bias > 15%`（過熱）= 10分；`close < MA_60` = 0分 |
| 中期籌碼 | 25 | `Inst_20D_Net` 為正 = 25分；為負 = 0分 |
| 背離扣分 | 15 | 無 `Price_Revenue_Divergence` = 15分；有背離 = 0分 |

### 3.3 價值評分（100分）

| 子項 | 權重 | 給分規則 |
| --- | --- | --- |
| 估值安全邊際 | 30 | `PE_Percentile < 20%` = 30分；`20%~40%` = 20分；`40%~60%` = 10分；`>60%` = 0分（若 `TTM_EPS_Valid=False`，此項直接以 `PB_Percentile` 取代計算） |
| 獲利能力穩定性 | 25 | `ROE_TTM > 15%` 且 `ROE_Stability` 屬於後段班（同產業內標準差較低，或簡化為標準差 < 5%）= 25分；`ROE_TTM > 10%` = 15分；否則 = 5分 |
| 毛利穩定性 | 20 | `Gross_Margin_Stability` 標準差 < 3% = 20分；`3%~6%` = 10分；`>6%` = 0分 |
| 財務結構 | 15 | `Debt_Ratio < 40%` = 15分；`40%~60%` = 8分；`>60%` = 0分 |
| 現金流覆蓋 | 10 | `FCF_vs_Dividend > 0` = 10分；否則 = 0分 |

**資料不足處理**：若 `Data_Years_Available < 5`，總分乘以 0.8 折扣，並在 UI 上註記樣本不足。

### 3.4 定存股評分（100分）

| 子項 | 權重 | 給分規則 |
| --- | --- | --- |
| 配息連續性 | 35 | `Dividend_Continuity_Years` 達到 `Data_Years_Available` 全年（無中斷）= 35分；中斷1次 = 15分；中斷2次以上 = 0分 |
| 配息率健康度 | 25 | `Payout_Ratio` 落在 50%~80% 區間且 `Payout_Ratio_Stability` 標準差 < 10% = 25分；落在區間但波動大 = 15分；超出區間（過高或過低）= 5分 |
| 現金流覆蓋 | 25 | `FCF_Coverage > 1.5` = 25分；`1.0~1.5` = 15分；`<1.0` = 0分（代表配息可能靠借款支應，警訊） |
| 財務結構趨勢 | 15 | `Debt_Ratio_Trend < 0`（負債比逐年下降或持平）= 15分；持平略升 = 8分；明顯上升 = 0分 |

**資料不足處理**：同價值股，`Data_Years_Available < 5` 時總分乘以 0.8，並註記。

## 四、給 Cline 實作的補充說明

1. **權重與門檻建議寫成 config**（例如 `core/scoring_config.py` 或一個 JSON/YAML），格式類似：
   ```
   SHORT_TERM_WEIGHTS = {"ma_alignment": 30, "volume": 25, "institutional": 25, "margin_health": 20}
   ```
   這樣未來你想調整「量能配合的門檻從1.5改成1.3」，不用改 scorer.py 的邏輯本體，只改設定值。

2. **每個子項給分函式應該獨立成小函式**（例如 `score_ma_alignment(row) -> int`），方便日後你想針對某一項單獨測試或調整，也方便我們之後討論 AI prompt 時直接引用「短線總分是怎麼組成的」這件事給模型參考。

3. **輸出時保留子項明細，不要只丟總分**。例如 `scorer.py` 回傳的結構建議是：
   ```
   {
     "short_term": {
       "total": 78,
       "breakdown": {
         "ma_alignment": 30,
         "volume": 15,
         "institutional": 25,
         "margin_health": 8
       }
     },
     ...
   }
   ```
   這樣前端「四維度分析卡」可以顯示子項雷達圖，AI 決策層（斷點9）也能拿到細項理由，而不是只有一個冷冰冰的分數，寫 prompt 時會更好發揮。

4. **驗收方式建議**：斷點7做完後，挑一支你認為明顯偏「價值型」的股票（例如高股息、低本益比的傳產股）跟一支明顯偏「短線活躍」的股票（例如當沖熱門的中小型股），跑一次四個分數，確認前者價值/定存分數明顯高於短線分數，後者相反。如果分數區分度不明顯，代表某個子項權重或門檻需要調整。
