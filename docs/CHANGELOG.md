## v2.1（2026-07-15）

### Phase 2：核心 8 子項重構 + 系統性 Bug 修正

| # | 項目 | 說明 |
|:---|:---|:---|
| 🔴 1 | 法人籌碼 20 日代理門檻 | `scoring_config.py` 新增隔離的 `inst_20d_surrogate_*` 門檻，不與 10 日共用 |
| 🔴 2 | 訊號重疊修正 | `score_catalyst` 改為「突破性事件」邏輯，與 `revenue_momentum` 定量階梯區隔 |
| 🔴 3 | RSI 重複扣分移除 | `apply_risk_modifier` 改為 `apply_all_modifiers`，RSI 過熱由 `score_volatility_risk` 子項處理 |
| 🟡 4 | 缺資料預設值（無罪推定） | NaN→0 分，部分子項無資料時→70 分中性值 |
| 🟡 5 | 統一 Modifier 疊加順序 | `apply_all_modifiers()` 整合 Data Quality × Industry Bias × Risk，`np.clip(0, 100)` |
| 🟡 6 | 補齊五級 _poor 檔位 | 所有 24 子項新增 `_poor` 門檻（0 分） |
| 🟢 7 | 常數搬移 | `FINANCE_SECTORS`, `FINANCE_STOCK_IDS`, `RSI_OVERHEAT_*` 搬至 `scoring_config.py` |

### 其他變更

- **還原股價模組**：新增 `data/price_adjuster.py`，使用 FinMind 官方事件資料（SplitPrice/CapitalReduction/ParValueChange）計算還原股價 `adj_close`
- **成交量還原**：`adj_volume` 支援分割/減資的張數校正（跨期量指標 `Vol_MA_5` 優先使用）
- **技術指標**：MA、Bias、MACD、High/Low、Dist_High、MA20_Slope 全改為使用 `adj_close`
- **雙軌四維投票**：`_handle_has_position` 同時跑積極(60/40) + 保守(70/50) 並取最保守動作
- **雙軌建議價**：priority 2（波段）、priority 3（短線）、priority 4-A（價值/定存）全部改為新版格式
- **快取管理**：`refresh_and_restart.bat` 一鍵清除 Parquet + SQLite + 重啟
- **文件重構**：`docs/` 拆分為 README / SETUP / ARCHITECTURE / SCORING / CHANGELOG

### Bug 修復

- **app.py 快取版本檢測**：`hasattr(old_ta, 'agg_entry')` 對 dataclass 永遠 True，改為檢查 `agg_entry is None` + 訊息內容
- **app.py 輸入庫存重撈 FinMind**：刪除 `cache.get("has_position", False)` 錯誤判斷邏輯
