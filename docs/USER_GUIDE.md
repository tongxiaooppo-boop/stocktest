## 1. 系統概述

本系統是一個**台股盤後分析工具**，整合了：

- **資料層**：透過 FinMind API 取得台股股價、財報、營收、法人籌碼、融資券等 12 種資料
- **評分引擎**：四種投資風格（短線、波段、價值、定存）獨立評分，短線支援雙分析師人格（激進型/穩重型），每種風格細分 6~8 個子項
- **持有建議**：雙軌四維投票機制 + 飛刀濾網 + 鐵盾防禦，產生買/賣/持有建議
- **歷史回測**：Walk-forward 回測，支援積極(60/40)和保守(70/50)雙策略，回測跟隨分析師選擇
- **AI 解說**：串接 DeepSeek API，像分析師一樣解說每筆進出點位的時空背景

> ⚠️ **盤後版**：資料來源為 FinMind 歷史資料庫，非即時報價。  
> 📊 股價收盤 14:30~15:30 更新｜🏦 三大法人 16:00~17:00 更新｜💰 融資券 20:00~21:30 更新

## 3. 第一步：申請 API 金鑰

### FinMind API Token（必填）

1. 前往 [FinMind 官網](https://finmindtrade.com) 註冊帳號
2. 登入後進 **Dashboard** → 左側選單 **API Token**
3. 複製那串 Token（`eyJ0eXAiOiJKV1Qi...` 開頭）
4. 貼到系統側邊欄的輸入框

### DeepSeek API Key（選填 — 啟用 AI 解說）

1. 前往 [DeepSeek Platform](https://platform.deepseek.com) 註冊
2. 建立 **API Key** 並複製
3. 貼到系統側邊欄的輸入框

> 💡 沒填 DeepSeek Key 也能分析，只是沒有 AI 解說功能。

## 4. 第二步：啟動系統

### 本地執行

```bash
pip install -r requirements.txt
streamlit run app.py
```

瀏覽器自動開啟 `http://localhost:8501`。

### 雲端部署（Render）

| 項目 | 設定值 |
|:---|:---|
| Build Command | `pip install -r requirements.txt` |
| Start Command | `streamlit run app.py --server.port $PORT --server.address 0.0.0.0` |
| 環境變數 | `FINMIND_TOKEN`, `DEEPSEEK_API_KEY`（選填） |

## 5. 第三步：輸入股票與分析

### 側邊欄設定

1. **API 金鑰** — 貼上 FinMind Token（必填）、DeepSeek Key（選填）
2. **股票代號** — 輸入 4 碼台股代號（如 `2330` 台積電）
3. **個人化持股**（選填）— 輸入持股均價與股數，系統會產生個人化持倉建議
4. **🧠 短線分析師**（v3.0 新功能）— 選擇分析師人格：
   - 🔥 **激進型**：重動能/慣性突破，適合強勢股追價
   - 🛡️ **穩重型**：重趨勢/法人籌碼，適合盤整或保守進場
5. 點擊 **「🔍 開始分析」**

### 分析師切換與快取

- 切換分析師人格後重新分析，系統**不會重拉 FinMind 資料**（v3.2），僅重新評分，大幅節省時間
- 不同分析師的評分結果各自獨立快取

### 進度提示

系統會顯示 5 段式進度條：

```
📥 撈取資料 → 🔄 建構母表 → 📊 計算指標 → 🎯 評分分析 → 🤖 AI 解說
```

約 30~60 秒完成（視網路速度與資料量而定）。分析完成後會**自動快取**，同一天再次查詢直接顯示結果。

## 6. 第四步：閱讀分析結果

分析完成後，畫面依序顯示瀑布流區塊：

### 6.1 大盤與個股資訊卡

- 加權指數、漲跌幅、總成交金額
- 個股股價、漲跌幅、成交量、本益比

### 6.2 短線面圖表

- **K線 + 均線**（收盤價、MA5/MA10/MA20，近 1 個月）
- **成交量長條圖**
- **RSI 指標**（含超買 80 / 超賣 20 門檻線）
- **法人買賣超 + 股價**（三大法人每日買賣超柱狀圖）
- **融資券變化 + 股價**

### 6.3 中長線面圖表

分三個 tab 頁：

| Tab | 權重最高項目 | 圖表內容 |
|:---|:---|:---|
| 🟠 **波段動能** | 營收動能 25% + 中期趨勢 20% + 籌碼趨勢 20% | 營收YoY+股價、股價+MA20/MA60、法人20日累計+股價 |
| 🔵 **價值成長** | 成長能力 30% + 獲利品質 20% | ROE+毛利率+股價、TTM EPS+營收YoY+股價 |
| 🟢 **定存安全** | 配息紀錄 25% + 配息品質 20% + 現金流 20% | 歷年股利、TTM FCF+負債比+股價、PE/PB+殖利率+股價 |

### 6.4 四維度雷達圖

**第一排**: 短線買 🔴、短線賣 🔴、波段買 🟠、波段賣 🟠  
**第二排**: 價值 🔵、定存 🟢

- 短線雷達圖顯示 **8 個子項**（v3.0 起，含慣性突破 + 籌碼密集區），波段/價值/定存顯示 6 個子項
- 子項顏色：🟢 ≥80 分、🟡 ≥60 分、🟠 ≥30 分、🔴 <30 分
- 點擊 **「查看細項」** 展開完整評分明細（原始數據 + 評分結果 + 調整因子）
- 買入/賣出雙軌評分各自獨立顯示，雷達圖標籤含權重百分比

### 6.5 持倉建議

根據四維度投票 + 飛刀濾網 + 鐵盾防禦，產出四種動作：

| 動作 | 說明 |
|:---|:---|
| 🟢 **買進** | 系統判斷適合建立新倉位 |
| 🟢 **加碼** | 已持有，系統判斷可增加部位 |
| 🟡 **持有** | 目前適合繼續持有，不急著賣 |
| 🔴 **賣出/減碼** | 系統判斷風險上升，建議出場或減碼 |
| ⚪ **觀望/不建議** | 觸發安全濾網，建議等待 |

買進/加碼時會顯示**雙軌建議價位區間**（積極價 / 保守價）。

### 6.6 AI 解說

（需輸入 DeepSeek API Key）

AI 像分析師一樣提供：
- **📝 總結** — 一句話結論
- **✅ 加分原因 Top3** — 各風格最強的子項與理由
- **❌ 扣分原因 Top3** — 各風格最弱的子項與理由
- **👥 適合族群** — 什麼類型的投資者適合這檔股票
- **⚠️ 風險提醒** — 需要注意的風險點
- **🔍 觀察重點** — 後續應該關注的指標

## 7. 第五步：回測分析

### 7.1 參數設定

點擊側邊欄 **「📊 回測分析」**，在回測頁面展開 **「⚙️ 回測參數設定」**：

| 參數 | 預設值 | 說明 |
|:---|:---:|:---|
| 回測起始日 | 1 年前 | 回測開始日期 |
| 回測結束日 | 今天 | 回測結束日期 |
| 輸出頻率 | 每日 | D=每日, W=每週, M=每月 |
| 雙彈夾 | 關 | 50/50 分批建倉策略 |
| 賣出評分 | 關 | 短線/波段獨立賣出評分（見 7.3） |

> **v3.2 新功能**：回測分析會跟隨當前選擇的分析師人格（激進型/穩重型），使用對應的短線權重計算歷史分數。

系統會**同時執行兩種策略**：

| 策略 | 買入門檻 | 賣出門檻 | 適合 |
|:---|:---:|:---:|:---|
| 📉 **保守 (70/50)** | ≥ 70 分 | < 50 分 | 穩健型投資者 |
| 📈 **積極 (60/40)** | ≥ 60 分 | < 40 分 | 積極型投資者 |

### 7.2 閱讀回測結果

執行完成後顯示：

#### A. 策略切換按鈕
- 保守 (70/50) / 積極 (60/40) 自由切換

#### B. 四維度分數走勢（3 張獨立圖表）
| 圖表 | 顏色 | 分數範圍 |
|:---|:---|:---|
| 🔴 短線評分走勢 | 紅線（買入）/ 暗紅虛線（賣出） | 0~100 |
| 🟠 波段評分走勢 | 橘線（買入）/ 巧克力虛線（賣出） | 0~100 |
| 🔵 價值 🟢 定存評分走勢 | 藍線 + 綠線 | 0~100 |

每張圖附有買入門檻（綠色虛線）和賣出門檻（紅色虛線）。

#### C. 價格走勢與買賣訊號
- 收盤價走勢線 + 各風格的買入（△）和賣出（▽）標記
- 若設有持股均價，顯示成本線

#### D. 策略績效總覽
- 短線、波段、價值、定存、綜合 五種策略的：
  - 交易筆數
  - 總報酬率（含已實現 / 持有中損益）
  - 勝率

#### E. 交易明細表
- 每筆交易的買入/賣出日期、價格、報酬率

#### F. CSV 下載
- 積極 60/40、保守 70/50
- 若啟用賣出評分：額外產生積極+賣出評分、保守+賣出評分 共 4 份 CSV

### 7.3 賣出評分雙軌模式

**賣出評分是 v2.2 的核心新功能**。

傳統回測中，短線和波段只有一個總分，買入和賣出都用同一個分數判斷。但實務上買入理由和賣出理由不應該相同 —— 買入看的是上漲潛力，賣出看的是風險訊號。

**啟用後的效果：**

| 項目 | 原本（買入評分） | 啟用雙軌 |
|:---|:---|:---|
| 短線買入 | `short_term_score`（買入評分） | 不變 |
| 短線賣出 | `short_term_score`（買入評分） | **`short_term_score_sell`（賣出評分）** |
| 波段買入 | `swing_score`（買入評分） | 不變 |
| 波段賣出 | `swing_score`（買入評分） | **`swing_score_sell`（賣出評分）** |
| 綜合賣出 | 買入評分 | 短線/波段使用賣出評分 |

**在分數走勢圖中**：啟用後會多一條虛線顯示賣出評分，與買入評分配對比較。

### 7.4 雙彈夾分批建倉

50/50 資金分配策略：
1. **第一發**：條件滿足時進場 50% 資金
2. **第二發**：視模式而定
   - 📉 **下跌加碼**：價格跌過門檻（預設 -8%）後進場剩下 50%
   - 📈 **順勢突破**：分數創新高（高於門檻+10分）後進場剩下 50%
3. **全數出場**：分數跌破賣出門檻時一次清倉

### 7.5 AI 回測解說

（需輸入 DeepSeek API Key）

點擊「解說保守策略」或「解說積極策略」，AI 會回傳：

1. **📝 一句話總結** — 整體回測成效
2. **📊 各風格績效分析** — 短線/波段/價值/定存/綜合 各自的績效評語
3. **📖 交易點位深度解說** — 像分析師一樣順著時間軸解說進出點位
4. **🔍 整體策略核心診斷** — 策略優缺點與改善建議

## 8. 第六步：新聞輿情

分析時自動抓取新聞並進行情緒分析：

- **綜合情緒統計**：歷史新聞數、平均情緒分數、偏多/偏空比例
- **情緒分佈條**：偏多 🔴 中性 ⚪ 偏空 🟢 佔比
- **最新新聞列表**：時間、來源、標題、情緒判定
- **歷史情緒趨勢圖**：新聞情緒分數隨時間變化，附偏多/偏空門檻線

支援查詢其他股票的新聞 — 輸入代號後按「更新新聞」即可。

## 9. 常見問題

### Q1: 為什麼分析結果還是舊的？

A: 快取機制。系統會在同一天內使用快取資料。若要強制重新撈取：
- 點側邊欄 **「🔄 強制刷新資料」**
- 或刪除 `data/cache/` 目錄下的 `.parquet` 檔案後重啟

### Q2: 為什麼回測跑不出來？

A: 請先按一次「🔍 開始分析」完成資料撈取和母表建構，回測需要母表資料才能執行。也可以直接修改回測參數後點「▶️ 執行回測」。

### Q3: 為什麼圖表是空的？

A: 可能的幾種原因：
1. FinMind Token 無效 — 檢查是否正確複製
2. 股票代號不正確 — 台股請輸入 4 碼
3. 該股票 FinMind 資料不足 — 如剛上市的新股可能資料量不夠
4. 網路問題 — FinMind 伺服器暫時不穩定

### Q4: CSV 下載的內容是什麼？

A: 回測的 `signal_history` 包含：
- 每天的分數（short_term_score, swing_score, value_score, dividend_score）
- 各風格的買賣訊號標記
- 技術指標（MA、PE、PB 等）
- 雙軌建議價區間（agg_low/agg_high, cons_low/cons_high）

### Q5: 賣出評分和買入評分有什麼不同？

A: 賣出評分是獨立計算的，使用不同的子項權重和門檻設定。買入評分重視上漲潛力（如趨勢結構、動能強度），賣出評分則更重視風險訊號（如波動風險、籌碼背離）。詳見 `docs/SCORING.md`。

### Q6: 激進型和穩重型分析師差在哪？（v2.1 權重更新）

A: 兩種人格使用相同的 8 個評分子項，但**權重分配不同**（v2.1 已更新以下權重）：

| 子項 | 激進型 買/賣 | 穩重型 買/賣 |
|:---|:---:|:---:|
| 動能強度 | 30% / 25% | 5% / 5% |
| 慣性突破 | 25% / 35% | 5% / 10% |
| 趨勢結構 | 10% / 10% | 35% / 35% |
| 成交量結構 | 20% / 15% | 10% / 10% |
| 法人籌碼 | 5% / 5% | 20% / 15% |
| 籌碼密集區 | 5% / 0% | 20% / 25% |
| 籌碼健康 | 0% / 0% | 5% / 0% |
| 波動風險 | 5% / 10% | 0% / 0% |

- 🔥 **激進型**：重動能/慣性，適合強勢股追價，買入捕捉突破、賣出察覺反轉
- 🛡️ **穩重型**：重趨勢/成本區/法人，適合盤整或保守進場，核心始終是趨勢結構與持有成本

### Q7: 切換分析師需要重新撈 FinMind 嗎？（v3.2）

A: **不需要**！v3.2 導入兩層快取機制，切換分析師時僅重新評分，股價/財報/籌碼等原始資料不重複呼叫 FinMind API，大幅節省時間與 API 額度。
