## 1. 主函式架構

**檔案位置**: `core/scorer.py` → `def score_swing(row, cagr_1_5y=None) -> dict`（第 845 行）

```
score_swing(row, cagr_1_5y)
    ├── score_revenue_momentum(row, cagr_1_5y)   # 權重 25%
    ├── score_mid_trend(row)                      # 權重 20%
    ├── score_institutional_trend(row)            # 權重 20%
    ├── score_earnings_growth(row)                # 權重 15%
    ├── score_valuation_position(row)             # 權重 10%
    └── score_catalyst(row)                       # 權重 10%
              ↓
       加權總分 (base_score)
              ↓
    apply_all_modifiers(base_score, row, "swing", dq_modifier, industry_bias)
              ↓
       最終波段總分 (0-100)
```

**權重設定檔**: `core/scoring_config.py` 第 119-126 行

```python
SWING_WEIGHTS = {
    "revenue_momentum": 0.25,
    "mid_trend": 0.20,
    "institutional_trend": 0.20,
    "earnings_growth": 0.15,
    "valuation": 0.10,
    "catalyst": 0.10,
}
```

**回傳格式**:

```python
{
    "total": int,
    "breakdown": {
        "revenue_momentum": int,
        "mid_trend": int,
        "institutional_trend": int,
        "earnings_growth": int,
        "valuation": int,
        "catalyst": int,
    },
    "details": {
        "revenue_momentum": {...},
        "mid_trend": {...},
        ...
    }
}
```

## 2. 評分流程總覽

```
【輸入】row (dict), cagr_1_5y (可選)
         │
         ▼
   6 個子項各自獨立評分 (0-100)
         │
         ▼
   加權計算 → base_score = Σ(子項分數 × 權重)
         │
         ▼
   apply_all_modifiers()
    ├── Data Quality Modifier × (0.70 ~ 1.00)
    ├── Industry Debt Bias (僅價值/定存風格使用)
    └── Risk Modifier: penalty(-10~-15) + bonus(+5)
         │
         ▼
   np.clip(0, 100) → 最終波段總分
```

## 3. 子項評分函式

### 3.1 營收動能（revenue_momentum）

**檔案**: `core/scorer.py` 第 524 行 `def score_revenue_momentum(row, cagr_1_5y=None)`

**評分方式**: 依有無 `cagr_1_5y` 分兩種加權

**有 CAGR 時**:
```
composite = YoY × 35% + MoM × 15% + 加速度 × 15% + 1.5Y-CAGR × 35%
```

**無 CAGR 時**:
```
composite = YoY × 50% + MoM × 25% + 加速度 × 25%
```

#### 子指標 1：營收年增率（Revenue_YoY）

| 值 | 評分 |
|:---:|:----:|
| ≥ 30% | 100 |
| ≥ 20% | 85 |
| ≥ 10% | 70 |
| ≥ 0% | 50 |
| < -20% | 0 |

#### 子指標 2：營收月增率（Revenue_MoM）

| 值 | 評分 |
|:---:|:----:|
| ≥ 5% | 100 |
| ≥ 2% | 85 |
| ≥ 0% | 70 |
| ≥ -5% | 50 |
| < -10% | 0 |

#### 子指標 3：營收加速度（Revenue_Accelerating）

| 值 | 評分 |
|:---:|:----:|
| True (1) | 100 |
| False (0) | 0 |
| NaN | 0 |

#### 子指標 4：1.5Y-CAGR（年複合成長率）

| 值 | 評分 |
|:---:|:----:|
| ≥ 15% | 100 |
| ≥ 8% | 85 |
| ≥ 0% | 70 |
| ≥ -8% | 50 |
| < -50% | 0 |

#### 門檻設定（scoring_config.py 第 129-147 行）

```python
"rev_yoy_excellent": 30,
"rev_yoy_good": 20,
"rev_yoy_normal": 10,
"rev_yoy_weak": 0,
"rev_yoy_poor": -20,
"rev_mom_excellent": 5,
"rev_mom_good": 2,
"rev_mom_normal": 0,
"rev_mom_weak": -5,
"rev_mom_poor": -10,
"rev_accel_true": 1,
"cagr_1_5y_excellent": 15,
"cagr_1_5y_good": 8,
"cagr_1_5y_normal": 0,
"cagr_1_5y_weak": -8,
"cagr_1_5y_poor": -50,
```

#### 例外處理

- `Revenue_YoY`、`Revenue_MoM` 為 NaN → 代入 0
- `Revenue_Accelerating` 為 NaN → `accel_score = 0`
- `cagr_1_5y` 為 None 或 NaN → 不計入，無 CAGR 加權方式

### 3.2 中期趨勢（mid_trend）

**檔案**: `core/scorer.py` 第 596 行 `def score_mid_trend(row)`

**評分方式**: MA20 斜率 + 站上均線判斷

| 條件 | 評分 | 說明 |
|:-----|:----:|:-----|
| MA20_Slope > 0 且 close > MA60 | **100** | 中期多頭（20MA 向上 + 站上年線） |
| MA20_Slope > 0 | **85** | 20MA 向上（短中期偏多） |
| close > MA60 | **70** | 股價在年線之上 |
| close > MA20 | **50** | 股價在月線之上 |
| close ≤ MA20 | **0** | 短期偏空 |

#### 變數說明

| 變數 | 資料來源 | 說明 |
|:-----|:---------|:-----|
| MA20_Slope | processor 計算（OLS 斜率） | 20 日移動平均線斜率 |
| close | 每日收盤價 | — |
| MA_20 | processor 計算 | 20 日移動平均線 |
| MA_60 | processor 計算 | 60 日移動平均線 |

#### 例外處理

- `close` 為 NaN → 回傳 score=0，note="close 資料不足"
- `MA20_Slope` 為 NaN → `ma20_up = False`（不判定斜率）
- `MA_20` 或 `MA_60` 為 NaN → 對應的站上判斷為 False

### 3.3 籌碼趨勢（institutional_trend）

**檔案**: `core/scorer.py` 第 648 行 `def score_institutional_trend(row)`

**評分方式**: OLS 斜率優先，20 日累計法人買超備援

#### 有 Inst_Slope_20D（斜率）時：

| 條件 | 評分 |
|:-----|:----:|
| Inst_Slope_20D ≥ 50,000 | 100 |
| Inst_Slope_20D ≥ 0 | 85 |
| Inst_20D_Net ≥ 0 | 70 |
| Inst_Slope_20D > -50,000 | 50 |
| 其餘 | 0 |

#### 無斜率時（Inst_20D_Net 備援）：

| 條件 | 評分 |
|:-----|:----:|
| ≥ 10,000,000 | 100 |
| ≥ 2,000,000 | 85 |
| ≥ 0 | 70 |
| ≥ -2,000,000 | 50 |
| < -10,000,000 | 0 |

#### 門檻設定（scoring_config.py 第 149-162 行）

```python
# 斜率門檻
"inst_trend_slope_excellent": 50000,
"inst_trend_slope_good": 0,
"inst_trend_slope_normal": -25000,
"inst_trend_slope_weak": -50000,
"inst_trend_slope_poor": -100000,
# 20 日累計備援門檻
"inst_20d_net_excellent": 10000000,
"inst_20d_net_good": 2000000,
"inst_20d_net_normal": 0,
"inst_20d_net_weak": -2000000,
"inst_20d_net_poor": -10000000,
```

#### 例外處理

- `Inst_20D_Net` 為 NaN → 回傳 score=0，note="Inst_20D_Net 資料不足"

### 3.4 獲利成長（earnings_growth）

**檔案**: `core/scorer.py` 第 707 行 `def score_earnings_growth(row)`

**評分方式**:

```
composite = TTM_EPS × 60% + EPS_YoY × 40%
```

#### 子指標 1：TTM EPS（近四季每股盈餘）

| 值 | 評分 |
|:---:|:----:|
| ≥ 15 | 100 |
| ≥ 8 | 85 |
| ≥ 3 | 70 |
| ≥ 0 | 50 |
| < -5 | 0 |

#### 子指標 2：EPS YoY（EPS 年成長率）

| 值 | 評分 |
|:---:|:----:|
| ≥ 30% | 100 |
| ≥ 15% | 85 |
| ≥ 5% | 70 |
| ≥ 0% | 50 |
| < -20% | 0 |

#### 門檻設定（scoring_config.py 第 164-174 行）

```python
"ttm_eps_excellent": 15,
"ttm_eps_good": 8,
"ttm_eps_normal": 3,
"ttm_eps_weak": 0,
"ttm_eps_poor": -5,
"eps_yoy_excellent": 30,
"eps_yoy_good": 15,
"eps_yoy_normal": 5,
"eps_yoy_weak": 0,
"eps_yoy_poor": -20,
```

#### 例外處理

- `TTM_EPS` 為 NaN 或 `TTM_EPS_Valid = False` → `eps_score = 0`
- `EPS_YoY` 不可用（denominator_invalid / insufficient_history） → `eps_yoy_score = 0`，含備註說明

### 3.5 估值位置（valuation）

**檔案**: `core/scorer.py` 第 762 行 `def score_valuation_position(row)`

**評分方式**: **反向評分** — PE/PB 百分位越低越好

```
composite = PE_Percentile(反) × 60% + PB_Percentile(反) × 40%
```

| 百分位 | 評分 | 說明 |
|:------:|:----:|:-----|
| ≤ 20% | 100 | 極低估值 |
| ≤ 40% | 85 | 偏低 |
| ≤ 60% | 70 | 中性 |
| ≤ 80% | 50 | 偏高 |
| > 95% | 0 | 極高估值 |

#### 門檻設定（scoring_config.py 第 177-186 行）

```python
"pe_percentile_excellent": 20,
"pe_percentile_good": 40,
"pe_percentile_normal": 60,
"pe_percentile_weak": 80,
"pe_percentile_poor": 95,
"pb_percentile_excellent": 20,
"pb_percentile_good": 40,
"pb_percentile_normal": 60,
"pb_percentile_weak": 80,
"pb_percentile_poor": 95,
```

#### 例外處理

- `PE_Percentile` 為 NaN → `pe_score = 70`（給中間分不扣分）
- `PB_Percentile` 為 NaN → `pb_score = 70`

---

### 3.6 催化因子（catalyst）

**檔案**: `core/scorer.py` 第 804 行 `def score_catalyst(row)`

**評分方式**: 突破性事件判定（v2.1 校準版）

| 條件 | 評分 |
|:-----|:----:|
| Revenue_Momentum ≥ 3（連續 3 月 MoM 成長） | 100 |
| Revenue_Momentum ≥ 1（持續加速成長） | 100 |
| 無 Revenue_Momentum 但 YoY > 30% | 85 |
| 其餘 | 70（中性不扣分） |

#### 例外處理

- `Revenue_Momentum` 與 `Revenue_YoY` 皆為 NaN → score=70（中性）

## 4. Modifier 疊加

與短線相同，統一透過 `apply_all_modifiers()`（第 120 行）處理。

### 4.1 Data Quality Modifier

| 資料年數 | Modifier |
|:--------:|:--------:|
| ≥ 8 年 | × 1.00 |
| ≥ 5 年 | × 0.95 |
| ≥ 3 年 | × 0.85 |
| < 3 年 | × 0.70 |

### 4.2 Risk Penalty

| 條件 | Penalty |
|:-----|:-------:|
| 負債比 > 70%（非金融業） | -10 |
| TTM_EPS < 0 | -15 |
| TTM_OCF < 0 且 Payout_Ratio > 100%（非金融業） | -15 |

### 4.3 Risk Bonus

| 條件 | Bonus |
|:----|:-----:|
| RSI_6 < 30 | +5 |
| 負債比 < 20%（非金融業） | +5 |

## 5. 例外處理

### 5.1 資料缺失（NaN）防禦

| 子項 | 防禦機制 |
|:-----|:---------|
| revenue_momentum | YoY/MoM/Accel 預設 0；cagr_1_5y 可選傳入 |
| mid_trend | close 為 NaN 跳過評分 |
| institutional_trend | Inst_20D_Net 為 NaN 跳過評分 |
| earnings_growth | EPS 有效性檢查；EPS_YoY 不可用給 0 分 |
| valuation | PE/PB 百分位 NaN 給 70 分（中性） |
| catalyst | 無資料給 70 分（中性） |

### 5.2 金融業處理

波段評分的 6 個子項均無金融業特殊跳過邏輯。金融業影響僅在 Modifier 層級。

```

                          ├─ 加速度          25%(無CAGR) / 15%(有CAGR)

institutional_trend 20%  ─┬─ Inst_Slope_20D (優先)
                          └─ Inst_20D_Net (備援)

earnings_growth     15%  ─┬─ TTM_EPS          60%
                          └─ EPS_YoY          40%

valuation           10%  ─┬─ PE_Percentile(反) 60%
                          └─ PB_Percentile(反) 40%

catalyst            10%  ── 突破性事件判定
─────────────────────────────────────────────────
合計             100%
```
