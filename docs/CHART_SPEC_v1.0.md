# 圖表規格書（診斷優先版）— 對應斷點 10-11

> 這份圖表清單的**首要目的是驗證資料撈取與計算正確性**，不是最終的美化版 UI。每張圖都標了「這張圖能幫你抓出什麼 bug」，等斷點4-9的數據都驗證過沒問題後，你可以再考慮要不要精簡或美化。先求對，再求美。
>
> 實作順序建議：先做「短線」這組（資料量小、驗證快），確認圖表框架跟資料串接沒問題後，再擴充其他三組。

## 五、使用建議

1. **每組最後一張「分數子項明細」圖是共用邏輯**，四組都一樣的元件（橫向長條圖，X軸是分數0-100，Y軸是子項名稱），可以在 `app.py` 寫成一個共用函式 `render_score_breakdown(style_name, breakdown_dict)`，四個地方呼叫，不用寫四次。

2. **標了「關鍵驗證圖」的那幾張，做完斷點11之後請優先人工核對**：
   - 波段圖2（月營收原始值）
   - 價值圖2（EPS負值標記）、圖3（TTM EPS公告日對齊）
   - 定存圖2（現金股利vs股票股利）、圖6（TTM_FCF公告日對齊）
   
   這幾張圖直接對應到斷點4驗收清單裡最容易出錯的幾個地方，等於是把「資料驗證」做成「看得到的圖表」，比對著CSV肉眼核對輕鬆很多。

3. **等資料都驗證沒問題後**，如果想精簡成更適合手機瀏覽的版面，可以考慮：短線保留1、3、5、6（4張）；波段保留1、3、5、7（4張）；價值保留1、4、7、8（4張）；定存保留1、3、4、8（4張）。但建議先用完整版跑過一輪資料驗證，確認沒問題後再精簡，不要一開始就跳過驗證圖。
