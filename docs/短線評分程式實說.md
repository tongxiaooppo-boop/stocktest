## 1. 主函式架構

**檔案位置**: `core/scorer.py` → `def score_short_term(row) -> dict`（第 474 行）

```
score_short_term(row)
    ├── score_trend_structure(row)    # 權重 20%
    ├── score_momentum(row)           # 權重 20%
    ├── score_volume_structure(row)   # 權重 20%
    ├── score_institutional(row)      # 權重 15%
    ├── score_chip_health(row)        # 權重 15%
    └── score_volatility_risk(row)    # 權重 10%
              ↓
       加權總分 (base_score)
              ↓
    apply_all_modifiers(base_score, row, "short", dq_modifier, industry_bias)
              ↓
       最終短線總分 (0-100)
```

**權重設定檔**: `core/scoring_config.py` 第 35-43 行

```python
SHORT_TERM_WEIGHTS = {
    "trend_structure": 0.20,
    "momentum": 0.20,
    "volume": 0.20,
    "institutional": 0.15,
    "chip": 0.15,
    "risk": 0.10,
}
```

**回傳格式**:

```python
{
    "total": int,           # 加權總分（經 Modifier 疊加前）
    "breakdown": {
        "trend_structure": int,
        "momentum": int,
        "volume": int,
        "institutional": int,
        "chip": int,
        "risk": int,
    },
    "details": {
        "trend_structure": {...},  # 各子項明細
        "momentum": {...},
        ...
    }
}
```

## 2. 評分流程總覽

```
【輸入】row (dict) — 單一股票/單日的各項技術面與籌碼面資料
         │
         ▼
   6 個子項各自獨立評分 (0-100)
         │
         ▼
   加權計算 → base_score = Σ(子項分數 × 權重)
         │
         ▼
   apply_all_modifiers()
    ├── Data Quality Modifier × (0.70 ~ 1.00)
    ├── Industry Debt Bias (僅價值/定存風格使用)
    └── Risk Modifier: penalty(-10~-15) + bonus(+5)
         │
         ▼
   np.clip(0, 100) → 最終短線總分
```

## 3. 子項評分函式

### 3.1 趨勢結構（trend_structure）

**檔案**: `core/scorer.py` 第 173 行 `def score_trend_structure(row)`

**評分方式**: 均線排列 × 60% + 站上均線數量 × 40%

#### 子指標 1：均線排列（MA_Alignment）

| 值 | 評分 | 條件 |
|:---:|:----:|:-----|
| ≥ 3 | 100 | 所有均線多頭排列（5>10>20>60） |
| ≥ 2 | 85 | 短天期均線多頭排列 |
| ≥ 1 | 70 | 短天期均線即將黃金交叉 |
| ≥ 0 | 50 | 均線糾結 |
| < 0 | 0 | 空頭排列 |

#### 子指標 2：站上均線數量（Above_MA_Count）

| 值 | 評分 | 條件 |
|:---:|:----:|:-----|
| = 4 | 100 | 站上 5/10/20/60 全站上 |
| ≥ 3 | 85 | 站上 3 條 |
| ≥ 2 | 70 | 站上 2 條 |
| ≥ 1 | 50 | 站上 1 條 |
| = 0 | 0 | 全數跌破 |

#### 門檻設定（scoring_config.py 第 45-56 行）

```python
"ma_alignment_excellent": 3,
"ma_alignment_good": 2,
"ma_alignment_normal": 1,
"ma_alignment_weak": 0,
"ma_alignment_poor": -1,
"above_ma_excellent": 4,
"above_ma_good": 3,
"above_ma_normal": 2,
"above_ma_weak": 1,
"above_ma_poor": 0,
```

#### 程式實作

```python
def score_trend_structure(row) -> dict:
    t = SHORT_TERM_THRESHOLDS
    
    # 均線排列 (MA_Alignment)
    ma_alignment = row.get("MA_Alignment", 0)
    ma_score = five_level_score(ma_alignment, {
        "_excellent": t["ma_alignment_excellent"],
        "_good": t["ma_alignment_good"],
        "_normal": t["ma_alignment_normal"],
        "_weak": t["ma_alignment_weak"],
    })
    
    # 站上均線數量
    above_count = 0
    for ma in ["MA_5", "MA_10", "MA_20", "MA_60"]:
        close = row.get("close", 0)
        ma_val = row.get(ma, 0)
        if pd.notna(close) and pd.notna(ma_val) and close > ma_val:
            above_count += 1
    above_score = five_level_score(above_count, {...})
    
    composite = ma_score * 0.6 + above_score * 0.4
    return {"score": int(round(composite)), "details": {...}}
```

#### 例外處理

- `MA_Alignment` 為 NaN → 代入 0（`row.get("MA_Alignment", 0)`）
- 任一均線值為 NaN → 該條均線不計入站上數量（`pd.notna(ma_val)` 防禦）

### 3.2 動能強度（momentum）

**檔案**: `core/scorer.py` 第 214 行 `def score_momentum(row)`

**評分方式**: RSI(6) + 距離高點比例 Dist_High_5D 雙條件（Phase 2 重構）

| 條件 | 評分 | 說明 |
|:-----|:----:|:-----|
| RSI(6) ∈ [65, 80] 且 Dist_High_5D ≥ -1.5% | **100** | 強勢動能，未過熱 |
| RSI(6) ∈ [50, 65) 且 MACD_HIST > 0 | **85** | 動能蓄積中 |
| RSI(6) ∈ [50, 80) | **70** | 中性偏強 |
| RSI(6) < 50 且 Dist_High_5D < -5% | **50** | 超賣反彈機會 |
| 跌破前 5 日最低點 或 RSI ≥ 85 | **0** | 破底或過熱 |
| 以上皆非 | **50** | 偏弱但未破底 |

#### 門檻與變數

| 變數 | 資料來源 | 說明 |
|:-----|:---------|:-----|
| RSI_6 | processor 計算 | 6 日相對強弱指標 |
| High_5D | processor 計算 | 前 5 日最高價 |
| Low_5D | processor 計算 | 前 5 日最低價 |
| MACD_HIST | processor 計算 | MACD 柱狀體（DIF - MACD） |

**Dist_High_5D 計算公式**:

```
Dist_High_5D = (close - max(high[-5:])) / close × 100%
```

#### 例外處理

- `RSI_6` 或 `close` 為 NaN → 回傳 score=0，note="RSI 或 close 資料不足"
- `High_5D` 為 NaN 或 ≤ 0 → `dist_high` 設為 None，跳過 Dist_High_5D 相關評分條件
- `Low_5D` 為 NaN → `break_low` 為 False（不判定破底）
- `MACD_HIST` 為 NaN → `macd_positive` 為 False

### 3.3 成交量結構（volume）

**檔案**: `core/scorer.py` 第 277 行 `def score_volume_structure(row)`

**評分方式**: Volume Ratio × 60% + 爆量程度 × 40%

#### 子指標 1：Volume Ratio（成交量比）

| 值 | 評分 | 條件 |
|:---:|:----:|:-----|
| ≥ 2.0 | 100 | 成交量是 5 日均量的 2 倍以上 |
| ≥ 1.5 | 85 | 明顯放大 |
| ≥ 1.2 | 70 | 微幅放大 |
| ≥ 1.0 | 50 | 正常水準 |
| < 0.5 | 0 | 量能不足 |

#### 子指標 2：爆量程度（Volume_Surge = volume / Vol_MA_5）

| 值 | 評分 | 條件 |
|:---:|:----:|:-----|
| ≥ 3.0 | 100 | 爆大量（3 倍以上） |
| ≥ 2.0 | 85 | 明顯爆量 |
| ≥ 1.5 | 70 | 略微爆量 |
| ≥ 1.0 | 50 | 正常 |
| < 0.5 | 0 | 量能萎縮 |

#### 門檻設定（scoring_config.py 第 58-68 行）

```python
"volume_ratio_excellent": 2.0,
```

**檔案**: `core/scorer.py` 第 375 行 `def score_chip_health(row)`

**評分方式**:

```
composite = Margin_5D_Change(反) × 40% + Short_5D_Change × 30% + SBL_5D_Change(反) × 30%
```

#### 子指標 1：融資 5 日變化（Margin_5D_Change）— **反向評分**

| 值（張數） | 評分 | 說明 |
|:----------:|:----:|:-----|
| ≤ -5 | 100 | 融資大幅減少（籌碼沉澱） |
| ≤ 0 | 85 | 融資減少 |
| ≤ 5 | 70 | 微幅增加 |
| ≤ 10 | 50 | 融資增加 |
| > 20 | 0 | 融資暴增 |

#### 子指標 2：融券 5 日變化（Short_5D_Change）— 正向評分

| 值（張數） | 評分 | 說明 |
|:----------:|:----:|:-----|
| ≥ 5 | 100 | 融券大幅增加（看空力道強） |
| ≥ 0 | 85 | 融券增加 |
| ≥ -5 | 70 | 微幅減少 |
| ≥ -10 | 50 | 融券減少 |
| < -20 | 0 | 融券大幅減少 |

#### 子指標 3：借券 5 日變化（SBL_5D_Change）— **反向評分**

| 值（張數） | 評分 | 說明 |
|:----------:|:----:|:-----|
| ≤ -100,000 | 100 | 借券大幅減少 |
| ≤ 0 | 85 | 借券減少 |
| ≤ 100,000 | 70 | 微幅增加 |
| ≤ 500,000 | 50 | 借券增加 |

### 3.6 波動風險（risk）

**檔案**: `core/scorer.py` 第 421 行 `def score_volatility_risk(row)`

**評分方式**: Bias_5D（5 日乖離率）+ RSI(6) 過熱判定（Phase 2 重構）

| 條件 | 評分 | 說明 |
|:-----|:----:|:-----|
| Bias_5D ∈ [-2%, 2%] 且 RSI ≤ 80 | **100** | 低波動安全 |
| Bias_5D ∈ [-5%, 5%] | **85** | 乖離適中 |
| Bias_5D > 10% 或 < -10% | **70** | 短線乖離過大 |
| 無過熱但乖離略大 | **50** | 偏弱但未破底 |
| RSI(6) > 85 | **0** | 觸發過熱扣分 |

> **v2.1 變更**：RSI 過熱扣分已移至本子項，`apply_all_modifiers` 中不再重複扣分。

#### 變數說明

| 變數 | 資料來源 | 說明 |
|:-----|:---------|:-----|
| Bias_5D | processor 計算 | (close - MA_5) / MA_5 × 100% |
| RSI_6 | processor 計算 | 6 日相對強弱指標 |

#### Fallback 機制

若 `Bias_5D` 為 NaN，自動嘗試取值 `MA60_Bias` 作為替代：

```python
if pd.isna(bias_5d):
    bias_5d = row.get("MA60_Bias", None)  # fallback to MA60_Bias
```

#### 例外處理

- `RSI_6` 為 NaN → 回傳 score=0，note="RSI 資料不足"
- `Bias_5D` 與 `MA60_Bias` 皆為 NaN → `bias_abs = 999`，跳過所有 Bias 相關條件

## 4. Modifier 疊加

**檔案**: `core/scorer.py` 第 120 行 `def apply_all_modifiers()`

### 疊加順序

```
最終分數 = (base_score × DQ_Modifier) - abs(Penalty) + Bonus
最終分數 = np.clip(最終分數, 0, 100)
```

### 4.1 Data Quality Modifier (DQ)

| 資料年數 | Modifier | 說明 |
|:--------:|:--------:|:-----|
| ≥ 8 年 | × 1.00 | 資料充足，無折扣 |
| ≥ 5 年 | × 0.95 | 微幅折扣 |
| ≥ 3 年 | × 0.85 | 中度折扣 |
| < 3 年 | × 0.70 | 大幅折扣，資料不可靠 |

### 4.2 Risk Penalty

| 條件 | Penalty | 說明 |
|:-----|:-------:|:-----|
| 負債比 > 70%（非金融業） | -10 | 負債過高 |
| TTM_EPS < 0 | -15 | EPS 為負值 |
| TTM_OCF < 0 且 Payout_Ratio > 100%（非金融業） | -15 | 盈餘品質不佳（現金流無法支撐配息） |

### 4.3 Risk Bonus

| 條件 | Bonus | 說明 |
|:----|:-----:|:-----|
| RSI_6 < 30 | +5 | RSI 超賣區 |
| 負債比 < 20%（非金融業） | +5 | 極低負債 |

## 5. 例外處理

### 5.1 資料缺失（NaN）防禦

| 子項 | 防禦機制 |
|:-----|:---------|
| trend_structure | `row.get("MA_Alignment", 0)` 預設 0；均線值 `pd.notna()` 檢查 |
| momentum | `RSI_6` 或 `close` 為 NaN 跳過評分，回傳 0 分 |
| volume | `Vol_MA_5` 為 NaN 或 ≤ 0 跳過爆量評分 |
| institutional | 所有欄位 `row.get(xxx, 0)` 預設 0 |
| chip | 所有欄位 `row.get(xxx, 0)` 預設 0 |
| risk | `RSI_6` 為 NaN 跳過評分；`Bias_5D` 為 NaN 嘗試 fallback |

### 5.2 金融業處理

短線評分的 **6 個子項均無金融業特殊跳過邏輯**（與價值/定存風格不同）。

金融業在短線評分中的影響僅限於：
- `apply_all_modifiers` 中的 Risk Penalty（負債比 > 70% 不扣分，由 `is_finance(row)` 判斷）
- `apply_all_modifiers` 中的 Risk Bonus（極低負債不加分，由 `is_finance(row)` 判斷）

### 5.3 `is_finance()` 判斷邏輯（第 23 行）

```python
def is_finance(row) -> bool:
    # 1. 產業名稱為「金融業」
    # 2. 股票代碼在 FINANCE_STOCK_IDS 清單中
    # 3. 股票代碼以 "28" 開頭且長度為 4 碼
```
