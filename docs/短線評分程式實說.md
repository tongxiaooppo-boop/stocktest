## 2. 評分流程總覽

```
【輸入】row (dict) — 單一股票/單日的各項技術面與籌碼面資料
         │
         ▼
   6 或 8 個子項各自獨立評分 (0-100)
         │
         ▼
   加權計算 → base_score = Σ(子項分數 × 權重)
         │
         ▼
   apply_all_modifiers()
    ├── Data Quality Modifier × (0.70 ~ 1.00)
    ├── Industry Debt Bias (僅價值/定存風格使用)
    └── Risk Modifier: penalty(-10~-15) + bonus(+5)
         │
         ▼
   np.clip(0, 100) → 最終短線總分
```

### 4.2 慣性突破/破壞（inertia_break）

**檔案**: `core/scorer.py` 第 414 行 `def score_inertia_break(row)`

#### 評分邏輯

| 分數 | 條件 |
|:---:|:---|
| 100 | 突破前10日高點 **且** 連漲 ≥3日 |
| 85 | 突破前10日高點 **或** 連漲 ≥3日 |
| 70 | 收盤價高於5日均線但無明顯突破 |
| 50 | 盤整（以上皆非） |
| 0 | 跌破前10日低點 **或** 連跌 ≥3日 |

#### 資料來源

| 欄位 | 說明 | 生產方式 |
|:---|:---|:---|
| `High_10D` | 近 10 日最高價 | `close.rolling(10).max()` |
| `Low_10D` | 近 10 日最低價 | `close.rolling(10).min()` |
| `Consec_Up_Days` | 連續上漲天數 | `close.diff().gt(0)` 分組累加 |
| `Consec_Down_Days` | 連續下跌天數 | `close.diff().lt(0)` 分組累加 |

> 以上全部使用 `TaiwanStockPrice`（Free）日K資料計算，不需額外 API。

### 4.3 籌碼密集區（chip_concentration）

**檔案**: `core/scorer.py` 第 451 行 `def score_chip_concentration(row, price_history)`

#### 運算原理

以多日收盤價 × 成交量的加權分布（histogram），取代單日盤中 POC（需要分K資料才算得出來），找出近 60 日內成交量最密集的價格區間（Chip Concentration Price, CCP）。

#### 計算步驟

1. 取近 60 日的 `close` 與 `volume`
2. 將收盤價區間等分為 30 個 bin
3. 每個 bin 累加該價格區間的總成交量
4. 找出成交量最大的 bin，其中心價即為 CCP
5. 計算 `Distance = (close - CCP) / CCP × 100`（%）

#### 評分表

| 分數 | Distance | 說明 |
|:---:|:---|:---|
| 100 | > 5% | 站穩在密集區上方，籌碼已消化 |
| 85 | 1% ~ 5% | 剛站上密集區 |
| 70 | -2% ~ 1% | 正在測試密集區 |
| 50 | -5% ~ -2% | 跌破密集區 |
| 0 | < -5% | 深跌破密集區，轉為壓力 |

#### 參數設定

| 參數 | 值 | 說明 |
|:---|:---:|:---|
| `lookback_days` | 60 | 統計區間（約一季） |
| `num_bins` | 30 | 價格分箱數 |
| `dist_excellent` | 5.0 | 站穩密集區 % 門檻 |
| `dist_good` | 1.0 | 剛站上密集區 % 門檻 |
| `dist_normal` | -2.0 | 測試密集區 % 門檻 |
| `dist_weak` | -5.0 | 跌破密集區 % 門檻 |

> 若 `price_history` 資料不足（< 10 筆），回傳中性分數 50，避免誤判為極端空頭。

#### 資料防呆

```python
vol_col = "volume" if "volume" in price_history.columns else (
    "Trading_Volume" if "Trading_Volume" in price_history.columns else "volume"
)
```

> fetcher 將 `Trading_Volume` 重命名為 `volume`，但仍保留原始名稱兼容。

### 4.4 雙分析師總分函式（score_short_term_by_profile）

**檔案**: `core/scorer.py` 第 496 行

```python
def score_short_term_by_profile(row, profile: str = "chaser", price_history: pd.DataFrame = None) -> dict:
    if profile not in STYLE_PROFILES:
        raise ValueError(f"未知的分析師人格: {profile}")

    # 動態 RSI 過熱門檻（均線護航機制）
    ma_alignment = row.get("MA_Alignment", 0)
    rsi_limit = 95 if pd.notna(ma_alignment) and ma_alignment >= 3 else 88

    # 計算 8 個子項
    trend = score_trend_structure(row)
    momentum = score_momentum(row, rsi_limit)
    volume = score_volume_structure(row)
    inst = score_institutional(row)
    chip = score_chip_health(row)
    risk = score_volatility_risk(row, rsi_limit)
    inertia = score_inertia_break(row)
    concentration = score_chip_concentration(row, price_history)

    sub_scores = {
        "trend_structure": ..., "momentum": ..., "volume": ...,
        "institutional": ..., "chip": ..., "risk": ...,
        "inertia_break": ..., "chip_concentration": ...,
    }

    buy_w = STYLE_PROFILES[profile]["buy"]
    sell_w = STYLE_PROFILES[profile]["sell"]
    buy_total = sum(sub_scores[k] * buy_w[k] for k in sub_scores)
    sell_total = sum(sub_scores[k] * sell_w[k] for k in sub_scores)

    return {
        "profile": profile,
        "total_buy": int(round(buy_total)),
        "total_sell": int(round(sell_total)),
        "breakdown": sub_scores,
        "details": {...}
    }
```

#### 回傳格式差異

| 欄位 | score_short_term (v2) | score_short_term_by_profile (v3) |
|:---|:---|:---|
| `total_buy` | ✅ | ✅ |
| `total_sell` | ✅ | ✅ |
| `profile` | ❌ | ✅ `"chaser"` / `"stable"` |
| `breakdown` | 6 keys | 8 keys（含 inertia_break + chip_concentration） |

## 5. v3.1 波段 catalyst 修正

### 問題

`score_catalyst()` 的 docstring 承諾「創12個月新高→100分」，但舊版程式碼從未實作此判斷，只檢查 `Revenue_Momentum` / `Revenue_YoY`，與 `revenue_momentum` 子項重複計分同一份原始資料。

### 修正

將 `catalyst` 改為獨立離散突破事件：

| 分數 | 條件 |
|:---:|:---|
| 100 | 最新單月營收創近12個月新高（突破性催化） |
| 85 | 最新單月營收創近6個月新高 |
| 75 | 未創高，但 Revenue_YoY > 30%（強勁成長） |
| 70 | 無明顯突破性催化因子（中性，不扣分） |

### 新增資料欄位

| 欄位 | 用於 | 生產位置 |
|:---|:---|:---|
| `Revenue_12M_High` | score_catalyst 12月創高判斷 | `processor.build_universal_base_table` |
| `Revenue_6M_High` | score_catalyst 6月創高判斷 | `processor.build_universal_base_table` |

```python
df_rev["Revenue_12M_High"] = df_rev["revenue"] >= df_rev["revenue"].rolling(12, min_periods=1).max()
df_rev["Revenue_6M_High"]  = df_rev["revenue"] >= df_rev["revenue"].rolling(6, min_periods=1).max()
```

> 使用 `>=` 避免浮點數比較誤差；`min_periods=1` 確保新股上市資料不足時仍可運算。

### 影響範圍

| 模組 | 是否變更 |
|:---|:---:|
| `SWING_BUY_WEIGHTS` | ❌ 不變 |
| `SWING_SELL_WEIGHTS` | ❌ 不變 |

| `score_revenue_momentum()` | ❌ 不變 |

與 v2.1 相同，參見 [SCORING.md](./SCORING.md) 例外處理章節。
